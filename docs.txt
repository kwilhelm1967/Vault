### Data Flow Narrative

**ONLINE PHASE (Purchase to Install):**

1. Customer visits website 
2. Stripe processes payment and triggers webhook
3. License server generates keys and creates download package
4. Brevo sends email with license keys + download link
5. Customer downloads installer from Linode server

**OFFLINE PHASE (After Install):**

1. App runs 100% locally - NO network calls
2. All vault data stored in browser localStorage
3. License activation makes exactly ONE server request the first time. After activation the app never calls any license API again (lifetime license ‚Äì fully offline).
4. Backups are manual file exports by user

### What Stays Online vs Offline

**Always Online:**

- Website/landing page 
- Stripe payment processing
- License server (Linode)
- Email delivery (Brevo)
- Download hosting

**Always Offline (After Install):**

- Password vault data
- Master password verification
- Encryption/decryption
- Export functionality
- All core app featurest - Complete Handover Documentation

## Executive Summary

Local Password Vault is a desktop password manager that operates **fully offline** after installation. The app uses React + TypeScript frontend with Electron for desktop functionality. Purchase and delivery flow uses Stripe, email delivery, and download hosting, but the core app **never calls home** after installation.

## 5) LICENSING SYSTEM ANALYSIS

### Current License Validation Process

1. **License Key Format**: `XXXX-XXXX-XXXX-XXXX` pattern
2. **Hardware Fingerprinting**: Generated from screen, system, and browser data
3. **Server Validation**: Calls `/api/validate-license` and `/api/activate-license`
4. **Local Storage**: Keys stored in browser localStorage
5. **Periodic Validation**: (REMOVED in lifetime mode ‚Äì no further server checks after first activation)

### Device Enforcement Limitations

**‚ùå CRITICAL ISSUE**: App cannot enforce device limits while truly offline.

**Current Behavior:**

- License validation requires internet connection
- Hardware fingerprinting can be bypassed
- No local cryptographic verification
- Users can share keys between devices if they go offline

**Marketing Statement Should Say:**
"License key activates once with an internet connection. After that, the app runs forever 100% offline on that device."

**Clarification:** Device limit enforcement is not possible offline; messaging should focus on trust + single activation.

### License Key Generation Script

**Location:** `server-api-examples/license-server.js` - `generateLicenseKey()` function
**Current Method:**

```javascript
function generateLicenseKey() {
  const chars = "ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789";
  const segments = [];
  for (let i = 0; i < 4; i++) {
    let segment = "";
    for (let j = 0; j < 4; j++) {
      segment += chars.charAt(Math.floor(Math.random() * chars.length));
    }
    segments.push(segment);
  }
  return segments.join("-");
}
```

### Recommended License Improvements

1. **Implement Cryptographic Signing**: Use Ed25519 or RSA signatures
2. **Embed Expiration Dates**: Include plan limits in signed payload
3. **Offline Validation**: Verify signatures locally without server calls
4. **Hardware Binding**: Stronger device fingerprinting

## 9) SUPPORT DOCUMENTATION & COMMON ISSUES

- **Check**: Internet connection during activation
- **Fix**: Generate new license key and revoke old one

**3. "Forgot master password" (20% of tickets)**

- **Reality**: NO PASSWORD RECOVERY POSSIBLE
- **Response**: "We cannot reset your master password as it's never stored on our servers. If you have a backup file and remember the password, you can restore from backup. Otherwise, you'll need to create a new vault."

**4. "Moving to new computer" (15% of tickets)**

- **Process**: Export vault ‚Üí Copy file to new computer ‚Üí Install app ‚Üí Import vault ‚Üí Reactivate license
- **License**: May need to reset device binding server-side

**5. "Antivirus blocking app" (10% of tickets)**

- **Common**: Windows Defender, Norton, McAfee false positives
- **Solution**: Add app to antivirus whitelist, verify installer hash
