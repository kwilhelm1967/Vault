## üõ†Ô∏è Development Environment Setup

### Prerequisites
- Node.js v18 or higher
- npm or yarn

### Initial Setup
```bash
# Clone the repository (if using Git)
git clone <repository-url>
cd local-password-vault

# Install dependencies
npm install
```

## üèóÔ∏è Building the Application

### Environment Configuration
1. Create environment files for different build modes:
   ```bash
   # Copy example environment files
   cp .env.test.example .env.test
   cp .env.prod.example .env.prod
   ```

2. Update the environment files with your configuration:
   - `.env.test` - For testing environment
   - `.env.prod` - For production environment

### Development Build
```bash
# Start development server (web only)
npm run dev

# Start development with Electron
npm run electron-dev

# Start in test mode
npm run start:test
```

### Production Build

#### Web Application
```bash
# Build for production
npm run build

# Build for production with production environment
npm run build:prod

# Preview production build
npm run preview
```

#### Desktop Applications

##### All Platforms
```bash
# Build for current platform
npm run dist
```

##### Windows (.exe)
```bash
# Build for Windows
npm run dist -- --win
```
Output:
- `release/LocalPasswordVault Setup.exe` (installer)
- `release/LocalPasswordVault-Portable.exe` (portable version)

##### macOS (.dmg)
```bash
# Build for macOS
npm run dist -- --mac
```
Output:
- `release/LocalPasswordVault.dmg`

##### Linux (.AppImage)
```bash
# Build for Linux
npm run dist -- --linux
```
Output:
- `release/LocalPasswordVault.AppImage`

## üì¶ Packaging for Distribution

### Creating Distribution Packages

#### Single User Package
```bash
# Create directory structure
mkdir -p dist-packages/single-user

# Copy application files
cp -r release/* dist-packages/single-user/

# Copy documentation
cp -r license-packages/single-user/* dist-packages/single-user/
cp README.txt dist-packages/single-user/

# Create ZIP archive
cd dist-packages
zip -r local-password-vault-single-user.zip single-user
cd ..
```
